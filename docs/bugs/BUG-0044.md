## 1. Bug Details

| Field | Value |
|---|---|
| **Ticket** | BUG-0044 |
| **Title** | Requestor never receives approval/refusal notifications |
| **Description** | In simple_api.py `take_approval_action()`, the code looks up the requestor with `User.query.filter_by(record_id=instance.rfpo.created_by)`. ## 2. Root Cause Analysis

The bug is a **field name confusion** between `RFPO.created_by` and `RFPO.requestor_id`:

- `RFPO.created_by` — `db.Column(db.String(64))` — stores a **display name** (e.g., `"John Bouchard"`)
- `RFPO.requestor_id` — `db.Column(db.String(32))` — stores a **User.record_id** (e.g., `"REQ001"`)

In `simple_api.py`, two functions use the wrong field to look up the requestor:

**1. `take_approval_action()` — line ~1406:**
```python
requestor = User.query.filter_by(
    record_id=instance.rfpo.created_by     # BUG: created_by is a display name, not a record_id
).first() if instance.rfpo.created_by else None
```

**2. `bulk_approval_action()` — line ~1632:**
```python
requestor = User.query.filter_by(
    record_id=inst.rfpo.created_by         # BUG: same issue
).first() if inst.rfpo.created_by else None
```

Since `User.record_id` never matches a display name string, the query always returns `None`. This causes:
- No email notification sent to the requestor
- No in-app notification created for the requestor (the `if requestor:` guard on line ~1414 prevents it)

**Correct pattern** (used elsewhere in the same codebase):
```python
# simple_api.py line 201-202 (correct)
requestor = User.query.filter_by(record_id=rfpo.requestor_id).first()

# custom_admin.py line 1035-1036 (correct)
requestor = User.query.filter_by(record_id=rfpo.requestor_id).first()

# simple_api.py line 3928-3929 (correct)
User.query.filter_by(record_id=rfpo.requestor_id).first()
```

**Related issue in admin panel:** `custom_admin.py` `_notify_workflow_complete()` (line ~1062) uses a different but also fragile pattern: `User.query.filter_by(email=rfpo.created_by)` with a fullname fallback. This may work coincidentally if the display name matches the fullname column, but is not reliable. This should also be fixed to use `requestor_id`.

## 3. Impact Assessment

**Affected files/functions:**
- `simple_api.py` — `take_approval_action()` (line ~1406-1408) and `bulk_approval_action()` (line ~1632-1634)
- `custom_admin.py` — `_notify_workflow_complete()` (line ~1062-1068) — related but less critical since admin panel is used less frequently

**User-facing impact:** **100% of RFPO requestors** are affected. No requestor has ever received an approval or refusal notification via the API workflow. Requestors must manually check their RFPO status to discover outcomes.

**Data impact:** No data corruption — notifications simply fail silently. The `requestor` variable evaluates to `None`, and all notification code is guarded by `if requestor:` checks.

**Scope:** Every RFPO that completes the approval workflow through the API (which is the primary workflow path). The admin panel has a separate code path with its own partial workaround.

## 4. Fix Strategy

### Phase 1: Core Fix (2 lines changed)
Replace `instance.rfpo.created_by` with `instance.rfpo.requestor_id` in both notification blocks:

**Fix in `take_approval_action()` (~line 1406):**
```python
# BEFORE (bug):
requestor = User.query.filter_by(
    record_id=instance.rfpo.created_by
).first() if instance.rfpo.created_by else None

# AFTER (fix):
requestor = User.query.filter_by(
    record_id=instance.rfpo.requestor_id
).first() if instance.rfpo.requestor_id else None
```

**Fix in `bulk_approval_action()` (~line 1632):**
```python
# BEFORE (bug):
requestor = User.query.filter_by(
    record_id=inst.rfpo.created_by
).first() if inst.rfpo.created_by else None

# AFTER (fix):
requestor = User.query.filter_by(
    record_id=inst.rfpo.requestor_id
).first() if inst.rfpo.requestor_id else None
```

### Phase 2: Fix Admin Panel Notification (Hardening)
Fix `_notify_workflow_complete()` in `custom_admin.py` to use the same reliable pattern:

```python
# BEFORE (fragile):
creator = User.query.filter_by(email=rfpo.created_by, active=True).first()
if not creator:
    creator = User.query.filter(User.active == True, User.fullname == rfpo.created_by).first()

# AFTER (correct):
creator = User.query.filter_by(record_id=rfpo.requestor_id, active=True).first() if rfpo.requestor_id else None
```

### Phase 3: Add Guard Comments (Documentation)
Add inline comments on the `RFPO.created_by` and `RFPO.requestor_id` fields in `models.py` to clarify their semantics and prevent future confusion:
```python
created_by = db.Column(db.String(64), nullable=False)  # Display name of creator (NOT a record_id — use requestor_id for lookups)
requestor_id = db.Column(db.String(32), nullable=False)  # User.record_id — use this for User lookups
```
