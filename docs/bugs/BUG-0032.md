## 2. Root Cause Analysis

The search bar was added in the FR-0001 implementation with conservative sizing:

```html
<div class="input-group input-group-sm" style="max-width: 320px;">
    <input type="text" name="search" class="form-control" ...>
```

Two issues compound to make the input feel too small:

1. **`input-group-sm`** — applies Bootstrap's small sizing, reducing font size to `0.875rem` and padding, making text harder to read
2. **`max-width: 320px`** — constrains the entire input group (input + search button + optional clear button) to 320px. After the buttons consume ~80px, the text input is only ~240px wide, which is cramped for search queries
3. **`form-select-sm`** on the status/priority dropdowns — same small sizing applied to all form controls in the toolbar

The fix is to remove all `sm` sizing classes from the search bar and dropdowns, and increase the `max-width` so the controls are at standard readable size.

## 4. Fix Applied

Two commits were needed to fully resolve the issue:

### Commit 1: `5472211` — Remove sm classes, widen max-width

In `templates/admin/tickets.html`:

**Before:**
```html
<div class="input-group input-group-sm" style="max-width: 320px;">
...
<select name="status" class="form-select form-select-sm" ...>
<select name="priority" class="form-select form-select-sm" ...>
```

**After:**
```html
<div class="input-group" style="max-width: 420px;">
...
<select name="status" class="form-select" ...>
<select name="priority" class="form-select" ...>
```

Changes:
- Removed `input-group-sm` — input uses standard Bootstrap form size
- Removed `form-select-sm` from both dropdowns — consistent height across all controls
- Increased `max-width` from `320px` to `420px`

### Commit 2: `a4be4a5` — Add min-width and flex-wrap

The first fix was insufficient — the flex layout (`d-flex gap-2`) still allowed the input group to be crushed to ~30px when the two dropdowns claimed their intrinsic width.

**Before:**
```html
<form class="d-flex gap-2" method="GET">
    <div class="input-group" style="max-width: 420px;">
```

**After:**
```html
<form class="d-flex flex-wrap gap-2" method="GET">
    <div class="input-group" style="min-width: 280px; max-width: 420px;">
```

Changes:
- Added `min-width: 280px` — prevents the search input from shrinking below a usable size
- Added `flex-wrap` on the form — lets the toolbar wrap to a second row on narrow viewports instead of crushing the input
